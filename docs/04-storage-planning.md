# 04 - Storage Planning

# Quick Summary

A clean folder structure is one of the most important parts of this project.

Instead of storing files randomly across the device, we will organize everything into dedicated directories. This improves productivity, makes automation easier, and keeps projects easy to maintain.

---

# Goal

After completing this chapter you will have:

* A dedicated development workspace
* Organized project folders
* Separate locations for AI models
* Reusable templates
* Prompt library
* Automation scripts
* Documentation storage

---

# Why Storage Organization Matters

Keeping files organized provides several benefits:

* Faster navigation
* Easier backups
* Better Git repositories
* Cleaner AI context
* Simpler automation
* Less duplicate data

---

# Recommended Root Directory

Create a new folder named:

```text
Developer
```

All development files used in this guide will be stored inside this folder.

---

# Recommended Folder Structure

Create the following directories:

```text
Developer/

Projects/
AI/
Models/
Prompts/
Templates/
Scripts/
Configs/
Docs/
Resources/
Downloads/
Workspace/
Archives/
Backups/
```

Do not worry if some folders remain empty for now.

They will be used in later chapters.

---

# Folder Purpose

## Projects

Stores all programming projects.

Examples:

* React
* React Native
* Node.js
* Python

---

## AI

Stores AI-related files.

Examples:

* Conversations
* Context files
* Memory files

---

## Models

Stores downloaded AI models when applicable.

Depending on the installation method, models may be stored elsewhere automatically.

---

## Prompts

Stores reusable prompts.

Examples:

* Bug fixing
* Refactoring
* Code review
* Documentation

---

## Templates

Stores reusable code templates.

Examples:

* React Components
* Express APIs
* Python scripts
* README templates

---

## Scripts

Stores automation scripts.

Examples:

* Update tools
* Git helpers
* Backup projects
* Cleanup storage

---

## Configs

Stores configuration files.

Examples:

* Git configuration
* Shell configuration
* Editor configuration

---

## Docs

Stores personal notes.

Examples:

* Architecture
* API documentation
* Design notes

---

## Resources

Stores useful reference material.

Examples:

* Cheat sheets
* PDFs
* Images
* Diagrams

---

## Workspace

Temporary working directory.

Files under active development can be placed here before moving them into permanent projects.

---

## Archives

Old projects that are no longer actively maintained.

---

## Backups

Manual backups before large changes.

---

# Naming Conventions

Use consistent folder names.

Recommended style:

```text
React-App

Node-API

Portfolio-Website

Expense-Tracker

Weather-App
```

Avoid:

```text
New Folder

Test123

Demo Final Latest

Temp
```

---

# Separate Personal Files

Keep development files separate from personal files.

Avoid mixing:

* Photos
* Videos
* Music
* Downloads

with programming projects.

---

# Storage Maintenance

Regularly:

* Delete unnecessary files
* Remove duplicate downloads
* Archive completed projects
* Remove unused AI models

A clean workspace improves productivity.

---

# Backup Strategy

Important projects should exist in at least one additional location.

Examples:

* GitHub
* External SSD
* Computer
* Cloud storage

Never rely on a single device.

---

# Verification

Confirm the following:

* Developer folder created
* All recommended folders created
* Projects folder available
* Prompts folder available
* Templates folder available
* Scripts folder available

If all folders exist, continue to the next chapter.

---

# Troubleshooting

## Folder Already Exists

Reuse the existing folder if it follows the recommended structure.

---

## Low Storage

Delete unnecessary downloads before continuing.

---

## Cannot Create Folder

Verify that storage permissions have been granted to your file manager.

---

# Optional Improvements

Advanced users may create additional folders:

```text
Logs/
Experiments/
Packages/
Datasets/
Testing/
Benchmarks/
```

These are optional and will not be required for this guide.

---

# Best Practices

* Keep folder names short.
* Use consistent capitalization.
* Avoid spaces when possible.
* Archive inactive projects.
* Keep active projects inside the Projects folder.
